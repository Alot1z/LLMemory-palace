# Task Brief 002: MCP Server Scaffold - Refined Architecture

## Context

Based on feedback, implementing refined v1.2.0 with:
- Reference-based minimal context (90% savings target, not "zero-token")
- Minimal tool set: init, scan, genome, resolve_ref, status
- Programmatic API usage (not CLI shelling)
- AUTO-MODE as separate layer
- Bun as first-class runtime

## Objective

Implement minimal MCP server that:
1. Exposes Palace core operations as MCP tools
2. Returns references + metadata instead of full content
3. Uses Palace programmatic API directly
4. Runs on both Node and Bun

## Scope

- Create mcp-server/ directory structure
- Implement 5 core tools (init, scan, genome, resolve_ref, status)
- Add reference generation and resolution
- Bun + Node dual runtime support

## Non-goals / Later

- AUTO-MODE integration (Task 003)
- patterns/flows/deps tools (v1.3.0)
- Full CLI wrapping (v1.3.0)

## Reference System

### Reference Format
```
palace://{type}/{hash}

Examples:
- palace://scan/7f3a8b2c
- palace://genome/9e1d4f6a
- palace://symbol/auth_login_5c3b
- palace://pattern/middleware_2a1f
```

### Reference Lifecycle
1. Generated by Palace semantic hash
2. Stored in .palace/references.json
3. Invalidated on file change (mtime check)
4. Auto-refresh on stale detection

### Depth Levels
- depth=1: Summary only (meta + count)
- depth=2: Outline (names + signatures)
- depth=3: Full content
